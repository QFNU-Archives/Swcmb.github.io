第二节：人以群居 物以类分——基本数据类型

[!TIP]

人以群居，物以类分。在C语言中，基本数据类型如同群居的人们，各有所归。

​ 你小时候分过类吗？比如：将玩具分为一类，将食物分为一类……分类，有利于更好的使用他们，在C语言中也是一样的。

​ 在C语言中，数据类型是用于定义变量或函数的返回值的数据种类。

​ 基本数据类型包括整数类型（int）、字符类型（char）、浮点数类型（float）和双精度浮点数类型（double）。这些类型可以表示不同的数值范围和精度。例如，整数类型可以表示整数值，字符类型可以表示单个字符，而浮点数类型可以表示带有小数部分的数值。

比如：

123456是int型

123.456000是float型

[!IMPORTANT]

浮点数类型（float）和双精度浮点数类型（double）在C语言中都是用于表示小数的数值类型，但它们在内存占用、精度和数值范围上有所不同。

  1. 内存占用：float类型在内存中占用4个字节，而double类型则占用8个字节。这意味着double类型可以存储更多的信息。
  2. 精度：float类型通常有8位有效数字，而double类型则有16位有效数字。这使得double类型在表示非常大或非常小的数值时，能够提供更高的精度。
  3. 数值范围：float类型的数值范围大约是-3.40E+38到+3.40E+38，而double类型的数值范围更大，因此它能够表示更大范围的数值。

这些差异使得double类型在需要更高精度和更大数值范围的情况下更为适用。

然而，使用double类型也会消耗更多的内存资源。

在实际编程中，选择使用哪种类型应根据具体需求和资源限制来决定。

(2)什么是顺序结构？

​ 顺序结构是编程中最基础的结构，它意味着程序会按照代码的书写顺序，从上到下逐行执行。

​ 这种结构就像我们日常生活中按部就班地完成任务。例如，做饭时，我们会先洗菜，再切菜，接着炒菜，最后盛菜上桌。这个过程中，我们按照一个固定的次序来操作，没有跳来跳去，也没有重复步骤。

​ 在顺序结构中，每一行代码都会按照它们在程序中出现的顺序一一被执行。这就像是早上起床后的一系列动作：先刷牙、再洗脸、然后吃早餐，每一步都是按照一定的顺序来的，不会跳过或者重复。

​ 以一个简单的例子来说明，如果我们有一个程序，它先给变量a赋值为3，然后给变量b赋值为5，最后交换这两个变量的值。这个过程就需要按照顺序来编写和执行代码，不能打乱这个顺序，否则最终的结果就会出现错误。

​ 总结来说，顺序结构就是程序按照代码的排列顺序，一步一步地执行，没有分叉，也没有回头路。它是编程中最基础的结构，让我们能够有条不紊地完成各种任务。

第二节：小箱子——变量、常量、输入输出

（1）变量

​ 我们在“基本数据类型”中提到：基本数据类型包括整数类型（int）、字符类型（char）、浮点数类型（float）和双精度浮点数类型（double）。相应的，变量也包括整数变量、字符变量、浮点数变量和双精度浮点数变量。

​ 变量可以被认为是代表可变数值的存储空间。通俗来讲，它就像一个容器，我们可以在其中存放数据，并且这个数据是可以改变的。以下是一些例子和解释，帮助理解变量是什么：

​ 可以将变量想象成一个有标签的盒子，盒子里的值（小球）可以随时更换，而标签则帮助我们识别这个盒子。例如，”年龄”是一个变量，因为它的值会随着时间的推移而变化，因为他是一个整数，我们通常使用int型存储它（盒子颜色），比如：

int xiao_ming_age//创建一个“盒子”（变量），盒子颜色是int"色"（int类型），盒子名（变量名）是xiao_ming_age（小明的年龄）
xiao_ming_age=10;//往这个盒子里面放一个球（值），为10

​ 在编程中，变量是基本的组成部分。它们是用于存储信息的内存位置的名称（盒子名称），这些信息可以在程序运行期间改变。通过变量名，我们可以访问和修改存储在该内存位置的值。

C 中的变量定义

变量定义就像是告诉计算机：“嘿，我需要一些空间来存储信息，请为我准备。” 具体来说，当你在编程时定义一个变量，你其实是在告诉计算机的“大脑”（也就是编译器）两件事情：

  1. 在哪里为我准备这个存储空间：这就像你要决定把东西放在家里的哪个抽屉或柜子里。计算机会在它的内存中找一个空位来存放这个变量。

  2. 这个存储空间要准备多大，以及用什么方式来存储：这就像你需要决定用多大的盒子来装东西，以及是否需要标签来标记它。数据类型就是告诉计算机这个盒子的大小和形状，比如是整数、浮点数还是字符等等。

所以，当代码中出现这样一行：“type variable_list;”，你其实是在说：“我想要一个可以存储type类型数据的存储空间，名字叫做variable_list。” 这样，计算机就知道要为这个叫做variable_list的变量预留出足够的空间，并且知道这个空间是用来存放type类型数据的。如下：

type variable_list;

type 是指变量的类型，就像给变量贴个标签，告诉我们它里面可以放什么种类的东西。这个类型可能是整数（比如年龄、计数）、浮点数（比如价格、重量）、字符（比如字母、符号）、或者更复杂的如指针（相当于其他数据的地址或引用）。我们甚至可以创建自己定义的复杂类型，像是游戏中的角色或银行账户这样的对象。

variable_list 指的是一系列变量名，它们之间用逗号**,**隔开，就像列一个清单一样。这些变量的名字是由字母、数字和下划线混合组成的，但是不能以数字开头，只能以字母或下划线开始。这就像是给孩子起名字，要遵守一定的规则，确保我们可以正确地叫出每个变量的名字，知道是哪个“人”。

[!CAUTION]

C语言中变量、常量和函数的命名需要遵循一些明确的规则，确保代码的合法性与可读性。这些规则包括：

  1. 组成字符：名称只能由字母、数字和下划线组成。
  2. 起始字符：名称必须以字母或下划线开头，不能以数字开头。
  3. 关键字：名称不能是C语言的关键字，即所谓的保留字。这些关键字被C语言用于特定的目的，如int、float等。
  4. 大小写敏感：C语言的命名是区分大小写的，例如ABC和abc会被视为两个不同的标识符。
  5. 描述性：虽然不是强制性的，但建议使用具有描述性的命名，以便清楚地表达变量、常量或函数的用途或类型。
  6. 长度限制：虽然C语言标准没有规定名称的长度上限，但大多数编译器允许的标识符长度通常为31个字符（某些编译器可能更长）。此外，为了提高代码的可读性和兼容性，应避免使用过长的名称。

下面列出几个有效的声明：

定义整型变量：

int age;//age 被定义为一个整型变量。
float salary;//salary 被定义为一个浮点型变量。
char grade;//grade 被定义为一个字符型变量。
int *ptr;//ptr 被定义为一个整型指针变量。

定义多个变量：

int i, j, k;//声明并定义了变量 **i、j 和 k**，这指示编译器创建类型为int的名为i、j、k的变量。
变量初始化

在 C 语言中，变量的初始化就是在你告诉计算机你要创建一个变量的时候，给这个新变量一个起始的值。你可以选择在声明变量的时候就给它一个初始值，或者在后面的代码里再给它赋值。

初始化变量的方式很简单，你只需要用等号（=）把变量名和它一开始的值连接起来，就像这样：

type variable_name = value;//类型 变量名 = 初始值；

这里的“类型”指的是你想让这个变量是什么类型，比如整数、浮点数还是字符等等。“变量名”就是你给这个变量取的名字，这样你后面就可以通过这个名字来使用或改变它。而“初始值”就是你想给这个新变量设置的起点数值。

举个例子，如果你想定义一个整数类型的变量叫做age，并且一开始就给它赋值为20，那么你可以写成：

int age = 20;

这里，int是数据类型，表示整型；age是变量名；20是初始值。这样，你就创建了一个名为age的变量，并且它一开始就有了值20。

下面列举几个实例：

int x = 10;         // 整型变量 x 初始化为 10
float pi = 3.14;    // 浮点型变量 pi 初始化为 3.14
char ch = 'A';      // 字符型变量 ch 初始化为字符 'A'
extern int d = 3, f = 5;    // d 和 f 的声明与初始化
int d = 3, f = 5;           // 定义并初始化 d 和 f
byte z = 22;                // 定义并初始化 z
后续初始化变量

在变量定义后的代码中，可以使用赋值运算符 = 为变量赋予一个新的值。

type variable_name;    // 变量定义
variable_name = new_value;    // 变量初始化

实例如下：

int x;          // 整型变量x定义
x = 20;         // 变量x初始化为20
float pi;       // 浮点型变量pi定义
pi = 3.14159;   // 变量pi初始化为3.14159
char ch;        // 字符型变量ch定义
ch = 'B';       // 变量ch初始化为字符'B'

在使用变量之前，我们需要给它一个初始值，这叫做初始化。如果没有初始化，变量就可能带着一些无意义的数字或信息，这些可能是电脑内存里以前留下的数据碎片。

想象你刚买了一个新的储物箱，并且打算用它来存放你的新书。在往里面放书之前，你会想要先清理一下，确保箱子是空的，没有旧的东西在里面。这样，你就可以确保当你把书放进去时，箱子里只有你想要的书，而不会有任何杂物。

程序里的变量也是这样，如果直接使用没有初始化的变量，你可能会遇到一些问题，比如程序不按预期工作或者崩溃。这是因为程序可能无法正确处理那些无意义的旧数据。

所以，就像整理你的储物箱一样，记得在使用变量前先清空（初始化）它，这样你就能确保它含有的是你需要的、有意义的数据了。

（2）常量

​ 常量就像是给某个固定的数值贴上了一个标签，在程序运行的时候这个标签背后的数字是不会变的。就像我们说的”巴黎是法国的首都”一样，不管我们说多少次，这个事实都不会改变。

常量可以是不同的类型，比如：

  • 整数常量就像是数字3、50或者100这样的固定数字。
  • 浮点常量就像是带小数点的数，比如3.14或者2.718这样不会变的数。
  • 字符常量就像是单个的字母’A’或者一个符号#这样不变的字符。
  • 字符串字面值就像是”hello”或者”world”这样的一串固定的文字。

使用常量的好处就是，当你需要多次使用同一个固定值时，你只需要用这个常量名就可以了，不需要记住那些具体的数字或者文字。而且，如果以后需要改变这个固定的数值，你只需要改一次定义的地方，其他用到这个常量的地方都会自动更新。

在写代码的时候，直接写出的数字、字符或者字符串通常都是常量，你也可以专门用关键字定义一个常量，比如在C语言里可以用#define或者const来定义一个常量，然后在代码的其他地方引用它。

定义常量

在C语言中，有两种简单的方式来定义常量：

  1. 使用#define预处理器：这就像是在程序里放一个“查找替换”的指令。比如，我们告诉程序，每次看到PI，就把它替换成3.14159。

  2. 使用const关键字：这就像是告诉程序，创建一个特殊的变量，这个变量的值一旦设置好了，就不能再改变。

#define预处理器的使用方式是这样的：

#define 常量名 常量值

例如，我们可以定义一个名为PI的常量，像这样：

#define PI 3.14159

在程序中使用这个常量时，编译器会自动把代码中的PI替换为3.14159。

const关键字就像是给一个值加上了一把锁，一旦你给它设了一个数，就不能再改变了。就像这样：

const int MAX_VALUE = 100;

这行代码就像是说：“我创建了一个名字叫做MAX_VALUE的箱子，并且在里面放了100这个数，然后给这个箱子加上了锁，所以里面的数就不能换了。”

所以，无论你在程序的哪个部分用到这个常量MAX_VALUE，它都会是100，你不能改变它。

const 声明常量要在一个语句内完成：

​ 请注意，把常量定义为大写字母形式，是一个很好的编程习惯。

​ 当我们在编程中使用常量（就是一个我们不打算改变的值）时，通常有一个不错的做法是将它们写成大写字母。就像这样：

const int MAX_VALUE = 100;

或：

#define PI 3.14159

​ 这样做的主要原因是，大写字母能很快地告诉我们和其他看代码的人：”这个是常量，它的值不会变”。这有助于避免以后不小心改变它的值，因为如果一个值被标记为常量，你就知道它应该是固定的。所以，保持这种习惯可以帮助我们更容易地理解和维护代码。

#define 与 const 区别

​ #define 和 const 都可以用来创建一些固定的值，但它们工作的方式有所不同。想象一下你在写一个故事（程序），其中有一些不会变的重要角色或物品（常量），你可以用两种不同的方式来描述它们：

  1. 替换机制：

    • 使用 #define 就像在故事开始前就用一个标签代替了某个角色，每次提到这个标签，你都会用同一个角色来替换。比如你定义了 “超级英雄” 代表 “有超能力的人”，那么每次说到 “超级英雄”，实际上就是在说 “有超能力的人”。
    • 使用 const 就像是给一个角色取了一个名字，并告诉大家他不会改变。比如你定义了一个常量 “超级英雄” 并给他赋予了一些特殊的属性，程序中的其他部分可以使用这个名字，但是这些属性是不可更改的。

  2. 类型检查：

    • #define 不关心类型，它只是一个简单的文字游戏，所以如果你不小心用错了，它也不会提醒你。
    • const 会告诉编译器这个常量是什么类型，如果其他地方使用不当，编译器会发出警告。

  3. 作用域：

    • #define 一旦被定义，在整个程序里都是有效的，直到你明确告诉它不再是那样。

    • const 定义的角色只在块级作用域内有效，出了这个块级作用域，它就不再是那个常量了。

      [!NOTE]

      在C语言中，”块级作用域”指的是在一个代码块（用一对花括号{}包围的部分）中定义的变量。这些变量只在该代码块内有效，一旦离开这个代码块，这些变量就不再存在或无法使用了。

  4. 调试和符号表：

    • 使用 #define 定义的常量在符号表中不会有相应的条目，因为它只是进行文本替换。

    • 使用 const 定义的常量会在符号表中有相应的条目，有助于调试和可读性。

      [!NOTE]

      C语言中的“符号表”就像是一本通讯录，它记录了程序中使用的所有名字（比如变量名和函数名）以及它们的相关信息，比如它们在内存中的位置、数据类型等信息。

      首先，我们来了解一下符号表的基本概念：

      • 符号表的目的：符号表的主要目的是帮助编译器和链接器管理程序中的各种标识符，确保它们在程序的不同部分之间能够正确地关联起来。
      • 符号表中的内容：符号表会存储每个标识符的名称、它在内存中的地址、它的数据类型（比如整数、浮点数或字符）、它的作用域（即在哪里可以被访问）以及其他一些属性。

      接下来，我们通过一个类比来理解符号表的作用：

      • 想象一下你正在组织一场大型活动，你需要记录下所有参与人员的名字、联系方式、分配给他们的任务等信息。这个记录就相当于符号表。通过这个记录，你可以快速找到每个人的信息，了解他们在哪里、做什么，以及如何联系他们。
      • 在C语言中，当你定义一个变量或函数时，就好像是给一个新来的参与者贴上标签并记录他们的信息。当程序运行时，如果需要访问某个变量或调用某个函数，符号表就会告诉程序这个标识符的地址，程序就可以找到它并进行操作。

      此外，符号表在编译过程中的作用也非常重要：

      • 在编译阶段，编译器会检查符号表中的每个标识符是否被正确声明和使用。如果有错误，比如一个变量未被声明就被使用，或者两个函数重名等情况，编译器就会报告错误。
      • 在链接阶段，链接器会使用符号表来解决不同编译单元之间的引用关系，确保所有的外部引用都能找到对应的定义。

      总的来说，符号表是C语言编程中不可或缺的组成部分，它确保了程序的正确编译、链接和运行。

​ 总的来说，虽然 #define 和 const 都可以帮你在程序中设定不变的元素，但 const 通常更受推荐，因为它提供了类型检查和更好的控制范围，这些特性对于编写清晰且容易管理的程序非常有用。
